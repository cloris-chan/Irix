## Windows Boundary

The Windows PoC separates target framework, CI toolchain check, and runtime minimum. Windows-targeted projects inherit `IrixWindowsTargetFramework=net10.0-windows10.0.26100.0` and `IrixWindowsSupportedOSPlatformVersion=10.0.15063.0` from `Directory.Build.props`; local builds rely on the .NET SDK to resolve a compatible installed Windows SDK such as 26100 or 28000 and report missing SDKs. The target framework stays on 26100 because .NET SDK 10.0.300 does not yet provide `Microsoft.Windows.SDK.NET.Ref` / TFM support for 28000, and CI verifies that Windows SDK 26100 or newer is installed instead of requiring one exact SDK directory. The 10.0.15063.0 runtime floor is intentional for PerMonitorV2 DPI awareness and display scale support. `IDWriteFactory4` is available within this runtime target and is the baseline DirectWrite factory for the glyph atlas path.

## Completed Context

- D3D12 GlyphAtlas text composition is the active Windows text path. DirectWrite/WIC remain source-data providers only.
- `D3D12DrawingBackend` and helper structs live in `Irix.Platform.Windows`.
- `TranslatorCore`, `TranslatorInput`, `TranslatorOutput`, and `TranslatorRetainedState` live in `Irix.Rendering`; `WindowDrawCommandTranslator` remains Poc glue.
- Glyph entries and atlas pages use stable value handles with generations; page reuse is format-scoped and retained-floor-gated.
- Text-cache allocation attribution exists as optional diagnostics. Further tree/layout/snapshot allocation work is paused until an ownership design names one target bucket.
- Retained partial apply and segmented render-source handoff exist behind explicit ownership/freshness/range guards and are enabled by default in the Poc runtime; `--no-partial-apply` remains the rollback switch. This is post-publication command/resource/hit-target reuse, not a `RenderPipeline.Build` layout skip.
- The color pipeline direction is written in [Color-Pipeline.md](Color-Pipeline.md): internal `Color` is now a 16-byte ideal linear BT.2020 / Rec.2020 straight-alpha value for style/property color, while the current active renderer remains SDR/sRGB and continues to use compact draw/window output payloads. HDR/scRGB/Rec.2100 output is deferred to a coordinated HDR backend/compositor implementation.
- Style, animation, and GPU composition contracts exist. Style remains internal IR rather than public UI authoring: the first internal semantic declaration layer maps width, height, background, foreground, opacity, translation, and control-state terms one-way into `VirtualNodeProperty` while keeping renderer processing names internal. Compact typed visual color properties can override draw-command payloads and stay visual-only; opacity/translation metadata has a separate composition mask, explicit style-delta work flags, and a preflight compiler for pure transform/opacity declarations. Public style/transition authoring preflight is documented and guard-covered: future UI authoring stays semantic, renderer processing names stay internal, and runtime owns transition start/cancel/retarget/commit before the compiler or diagnostics can participate. The Poc-owned style transition runtime path exists: runtime decisions flow through `StyleTransitionRuntimeCoordinator`, retained snapshots gate compositor install, start/retarget compile to transform/opacity declarations, and cancel/commit clear transform/opacity presentation. The first Counter app/control integration and lifecycle preflight slice also exists: `CounterStyleTransitionBridge` maps a single button `OwnershipSnapshot` delta through semantic transform/opacity declarations, typed active-scroll and multi-target outcomes stay on normal app dispatch, and `CounterStyleTransitionRuntimeBridge` waits for app render publication before coordinator install. Multi-target/active-scroll fallback now has an explicit abort-and-dispatch boundary: normal dispatch clears the active style transition presentation and tracker state before publishing app-owned state. `StyleTransitionCompletionTracker` is now a Poc-owned marker-driven completion tracker with an owner table keyed by `StyleTransitionOwnerKey`, and `StyleTransitionCompletionPump` connects it to the current single-owner Counter main-app path: `Once` transform/opacity transitions are decorated with a completion marker, compositor ticks publish that marker, the pump drains it, and the runtime coordinator applies the explicit `Commit` decision without adding a public or generic scheduler. Pump lifecycle diagnostics now make no-active, tick-rendered, completion-committed, tick-unavailable, error states, and active owner count observable through an internal Poc-owned snapshot/formatter. The true concurrent multi-owner transition preflight now includes Poc-owned batch/owner value types, deterministic acceptance/rejection tests, coordinator `ValidateBatch` coverage, validation-only `ValidateBatchRuntime` requirements pairing, rendering-owned presentation-set validation/conflict reporting for duplicate layer ids and overlapping command ranges, rendering-owned activation preflight/plan-set data, and owner-table completion tracking, all while reporting `PresentationStateChanged=false`; actual multi-owner execution and active presentation-set compositor state remain deferred. The active composition implementation direction is D3D12-backed transform/opacity, fixed-clip scroll presentation, retained target resolution, marker event publication, multi-layer composition execution, active hit-test remapping, main-app scroll presentation production, compositor-thread scroll retarget sample-and-cancel, reason-typed lifecycle invalidation/cancellation, and D3D12 layer content caching. Recent hardening covers fixed-clip/nested/sibling-layer hit-test remapping, mapped/unmapped marker runtime dispatch including cross-batch pump accounting, empty sample-and-cancel without cancellation, superseded/sample-and-cancel scroll-presentation idle waiters, explicit/render-invalidation/dispose idle waiter release, invalidation-after-explicit-cancel non-double-counting, lifecycle stale queued tick suppression, scroll marker cancellation queue cleanup, and fixed-clip/interleaved-paint-order layer-cache behavior.
- The scroll/input runtime owner boundary is written in `Poc-Promotion-Contracts.md`: logical scroll and input ownership belong to app/control runtime; layout feedback, compositor sampling, hit-test service access, input action mapping, and app message dispatch each require explicit adapter contracts before code moves. The first Poc-owned feedback sink, max-scroll feedback dispatch mapper, hit-test service, input action mapper, input-route app-message dispatch mapper, wheel-input scroll presentation dispatch sink, app runtime dispatch sink, and marker-runtime dispatch-through-sink cuts exist. Marker runtime dispatch source guards now pin the app-owned mapper/sink boundary without extracting Poc runtime.
- Viewport-space internal offscreen caching was removed from the mainline. Secondary paths are reserved for written blockers.

## Explicit Non-Goals

- Final text composition stays on the D3D12 GlyphAtlas path.
- No unguarded glyph script/format coverage expansion; new coverage needs matching oracle/regression coverage.
- No entry LRU or sub-rect free-list implementation before the ownership contract exists.
- No pixel oracle or DirectWrite layout renderer as a blocker for current D3D12 work.
- No retained-array, snapshot, or tree-builder allocation optimization without an ownership design plus attribution.
- No compatibility-driven public API expansion or code migration from `Irix.Poc` without a written contract.
- No compositor/runtime code extraction before the style, animation, and composition contracts are ready.
- No generic CPU/compatibility compositor as the first implementation route; use existing normal frame rendering as the explicit secondary path only when the D3D12-first path hits a written blocker.
- No Vulkan/Metal backend implementation in the current worklist.
- No HDR color output implementation in the current worklist; keep the active renderer on SDR/sRGB until the HDR backend/compositor output mapping slice is selected.
